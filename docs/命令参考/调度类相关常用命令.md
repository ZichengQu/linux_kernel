# 一、函数与接口整理

## 普通进程调度（CFS，普通 time-sharing）

| 使用场景 / 调度类 | 接口 / 命令                                     | 功能                             | 示例                                                                        |
| ---------- | ------------------------------------------- | ------------------------------ | ------------------------------------------------------------------------- |
| 普通进程（CFS）  | `nice(int inc)`                             | 调整当前进程的 nice 值（只能相对调整，不能直接获取值） | C语言：`nice(5);` <br>命令行：`nice -n 10 ./a.out`                               |
| 普通进程（CFS）  | `getpriority(int which, int who)`           | 获取 nice 值（可获取进程 / 进程组 / 用户范围）  | C语言：`getpriority(PRIO_PROCESS, pid);` <br>命令行：`ps -o pid,ni,cmd -p <pid>` |
| 普通进程（CFS）  | `setpriority(int which, int who, int prio)` | 设置 nice 值（支持进程 / 进程组 / 用户）     | C语言：`setpriority(PRIO_PROCESS, pid, 5);` <br>命令行：`renice -n 5 -p <pid>`   |

## 实时进程调度（SCHED\_FIFO / SCHED\_RR）

| 使用场景 / 调度类 | 接口 / 命令                                                             | 功能                                           | 示例                                                                                                                            |
| ---------- | ------------------------------------------------------------------- | -------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| 实时进程（RT）   | `sched_setscheduler(pid, policy, param)`                            | 设置调度策略（如 SCHED\_FIFO、SCHED\_RR）和 rt priority | C语言：<br>`c struct sched_param p={.sched_priority=50}; sched_setscheduler(pid, SCHED_FIFO, &p);` <br>命令行：`chrt -f -p 50 <pid>` |
| 实时进程（RT）   | `sched_getscheduler(pid)`                                           | 获取进程调度策略                                     | C语言：`sched_getscheduler(pid);` <br>命令行：`chrt -p <pid>`                                                                        |
| 实时进程（RT）   | `sched_getparam(pid, &param)` / `sched_setparam(pid, &param)`       | 获取/设置已有调度策略进程的参数（主要是 rt priority）            | C语言：`c struct sched_param p; sched_getparam(pid, &p);` <br>命令行：`chrt -f -p 50 <pid>`                                                                       |
| 实时进程（RT）   | `sched_get_priority_max(policy)` / `sched_get_priority_min(policy)` | 获取某调度策略的最大/最小优先级                             | C语言：`sched_get_priority_max(SCHED_FIFO);` <br>命令行：`chrt f5-m`                                                                                    |

---

## Deadline 调度（SCHED\_DEADLINE）

| 使用场景 / 调度类       | 接口 / 命令                                 | 功能                                 | 示例                                                                                                                                                                                      |
| ---------------- | --------------------------------------- | ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 周期实时进程（Deadline） | `sched_setattr(pid, attr, flags)`       | 设置 deadline 调度参数（周期、deadline、运行时间） | C语言：<br>[实际代码参考](https://github.com/ZichengQu/linux_kernel/blob/main/sched_setattr%E7%94%A8%E6%B3%95%E7%A4%BA%E4%BE%8B%20-%20%E8%87%AA%E5%AE%9A%E4%B9%89se-%3Eslice.c)</br><br>`c struct sched_attr attr = { .size=sizeof(attr), .sched_policy=SCHED_DEADLINE, .sched_runtime=10e6, .sched_deadline=30e6, .sched_period=30e6 }; sched_setattr(pid, &attr, 0);` <br>命令行<br>`chrt -d -p --sched-runtime 10000000 --sched-deadline 30000000 --sched-period 30000000 <pid>`|
| 周期实时进程（Deadline） | `sched_getattr(pid, attr, size, flags)` | 获取 deadline 调度参数                   | C语言：`sched_getattr(pid, &attr, sizeof(attr), 0);` <br>命令行：`chrt -p <pid>`|

## 其他调度相关接口

| 使用场景         | 接口 / 命令                                      | 功能                   | 示例                                                                                                                                                                  |
| ------------ | -------------------------------------------- | -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **CPU 亲和性**  | `sched_setaffinity(pid, cpusetsize, cpuset)` | 绑定线程运行在指定 CPU        | C语言：<br>`c cpu_set_t mask; CPU_ZERO(&mask); CPU_SET(0, &mask); sched_setaffinity(pid, sizeof(mask), &mask);` <br>命令行：`taskset -c 0-3,5 ./a.out` (启动时绑定 CPU 核心) 或 `taskset -cp 0,2,4,6-8 <pid>` (运行中修改 CPU 绑定) |
| **CPU 亲和性**  | `sched_getaffinity(pid, cpusetsize, cpuset)` | 获取线程 CPU 亲和性         | C语言：<br>`c cpu_set_t mask; sched_getaffinity(pid, sizeof(mask), &mask);` <br>命令行：`taskset -p <pid>` 或 `taskset -cp <pid>` 以十进制列表形式查看                                                               |
| **查看进程信息** | `/proc/$PID/sched` | 查看指定进程在 Linux 调度器（CFS）中的详细调度状态和运行统计信息，由内核函数 sched_show_task() 生成 | cat /proc/10997/sched |

## 二、从“调度器视角”看各类调度接口

| 分类    | 接口                    | 是否睡眠 | 是否立即切走         | 典型语义 |
| ----- | --------------------- | ---- | -------------- | ---- |
| 强制睡眠  | `schedule()`          | ✅    | ✅              | 我不跑了 |
| 主动退让  | `sched_yield()`       | ❌    | ✅              | 我让一下 |
| 条件退让  | `cond_sched()`        | ❌    | 视 need_resched | 文明让  |
| 定时睡眠  | `schedule_timeout()`  | ✅    | ✅              | 定点休眠 |
| 锁安全退让 | `cond_resched_lock()` | ❌    | 视情况            | 防死锁  |
| 用户态睡眠 | `nanosleep()`         | ✅    | ✅              | 节流   |

# 三、命令行工具汇总（补全示例）

| 命令                | 功能                    | 示例                                              |
| ----------------- | --------------------- | ----------------------------------------------- |
| `nice`            | 启动进程时指定 nice 值        | `nice -n 10 ./a.out`                            |
| `renice`          | 修改运行中进程的 nice 值       | `renice -n 5 -p 1234`                           |
| `chrt`            | 设置/查看实时调度策略和优先级       | `chrt -f -p 80 1234`                            |
| `taskset`         | 设置/查看 CPU 亲和性         | `taskset -c 0,1 ./server` <br>`taskset -p 1234` |
| `schedtool`（额外工具） | 高级调度控制（支持 deadline 等） | `schedtool -E -t 30000:10000 -p 1234`           |

# `chrt` 命令选项说明

| 选项 (短格式) | 选项 (长格式)          | 说明 |
| :------------ | :--------------------- | :--- |
| **策略选项**  |                        |      |
| `-b`          | `--batch`              | 设置策略为 `SCHED_BATCH` |
| `-d`          | `--deadline`           | 设置策略为 `SCHED_DEADLINE` |
| `-f`          | `--fifo`               | 设置策略为 `SCHED_FIFO` |
| `-i`          | `--idle`               | 设置策略为 `SCHED_IDLE` |
| `-o`          | `--other`              | 设置策略为 `SCHED_OTHER` |
| `-r`          | `--rr`                 | 设置策略为 `SCHED_RR` (默认) |
| **调度选项**  |                        |      |
| `-R`          | `--reset-on-fork`      | 设置 reset-on-fork 标志 |
| `-T`          | `--sched-runtime <ns>` | 为 `SCHED_DEADLINE` 策略设置运行时 (runtime) 参数 (纳秒) |
| `-P`          | `--sched-period <ns>`  | 为 `SCHED_DEADLINE` 策略设置周期 (period) 参数 (纳秒) |
| `-D`          | `--sched-deadline <ns>`| 为 `SCHED_DEADLINE` 策略设置截止时间 (deadline) 参数 (纳秒) |
| **其他选项**  |                        |      |
| `-a`          | `--all-tasks`          | 对给定 PID 的所有任务（线程）进行操作 |
| `-m`          | `--max`                | 显示有效优先级的最小和最大值 |
| `-p`          | `--pid`                | 对现有的指定 PID 进行操作 |
| `-v`          | `--verbose`            | 显示状态信息 |
| **基础选项**  |                        |      |
| `-h`          | `--help`               | 显示此帮助信息 |
| `-V`          | `--version`            | 显示版本信息 |
